Java I/O Stream
2024-10-15 17:40:41 #Java

1. I/O流

在 Java 中，I/O 流（Input/Output Streams） 是用于处理数据读写的基础工具之一。无论是从文件中读取数据、向网络发送请求，还是从控制台获取输入，I/O 流都是 Java 程序与外部数据源进行交互的核心方式。

1.1 I/O 流分类

根据处理的数据单位不同分为：字节流 与 字符流

字节流用于处理二进制数据，即以字节为单位读写数据。它适用于处理任何类型的数据，包括文本、图像、音频、视频等文件，尤其是当我们需要处理非文本文件时（如图像、音频、视频等），字节流是首选。Java 中的字节流类都继承自 InputStream 和 OutputStream 类。

主要字节流类：

  • InputStream：字节输入流的基类，用于从源读取字节数据。
  • OutputStream：字节输出流的基类，用于将字节数据写入目的地。

字节流分类：

  • 输入字节流：
    • FileInputStream：从文件中读取字节数据。
    • ByteArrayInputStream：从字节数组中读取字节数据。
    • BufferedInputStream：为输入流提供缓冲，提高效率。
  • 输出字节流：
    • FileOutputStream：将字节数据写入文件。
    • ByteArrayOutputStream：将字节数据写入字节数组。
    • BufferedOutputStream：为输出流提供缓冲，提高效率。

字符流用于处理文本数据，即以字符为单位读写数据。字符流能够自动处理字符编码问题（如 UTF-8、UTF-16 等），适用于读取和写入文本文件。字符流通过内部的缓冲机制，将多个字节组成字符来进行操作。Java 中的字符流类都继承自 Reader 和 Writer 类。

主要字符流类：

  • Reader：字符输入流的基类，用于从源读取字符数据。
  • Writer：字符输出流的基类，用于将字符数据写入目的地。

字符流分类：

  • 输入字符流：
    • FileReader：从文件中读取字符数据。
    • BufferedReader：为字符输入流提供缓冲，提高效率，同时支持按行读取。
    • InputStreamReader：将字节流转换为字符流，常用于处理编码转换。
  • 输出字符流：
    • FileWriter：将字符数据写入文件。
    • BufferedWriter：为字符输出流提供缓冲，提高效率。
    • OutputStreamWriter：将字符流转换为字节流，常用于处理编码转换。

字节流与字符流的区别

比较点 字节流（Byte Stream） 字符流（Character Stream）
数据处理单位 以字节为单位（8 位） 以字符为单位（16 位）
适用场景 处理二进制数据，如图像、音频、视频等 处理文本数据，如文本文件、字符数据等
基类 InputStream 和 OutputStream Reader 和 Writer
常见类 FileInputStream、FileOutputStream FileReader、FileWriter
编码处理 不处理字符编码问题 自动处理字符编码问题
常用场景 处理非文本文件，如图片、音频、视频 处理文本文件，如 .txt 文件

1.2 FIS/FOS 操作

1.2.1 read()

功能：read() 方法用于从输入流中读取一个字节（8 位），返回值是一个整数。

返回值：

  • 返回下一个字节的数据，范围是 0 到 255（对应的 int 值），可以转换为字符。
  • 如果到达文件末尾（EOF），则返回 -1。

阻塞：如果没有可用的输入，read() 方法将阻塞，直到有数据可读或发生 I/O 错误。

抛出异常：如果发生 I/O 错误，将抛出 IOException。

1.2.2 read(byte[])

功能：read(byte[] b) 方法用于一次性从输入流中读取多个字节并将它们存储到指定的字节数组 b 中。与只读取单个字节的 read() 方法相比， 能够提升读取的效率。

参数：byte[] b：一个字节数组，用于存储从输入流中读取的数据。

返回值：

  • 如果读取成功，返回的是存储在字节数组 b 中实际读取到的字节数。
  • 如果到达文件末尾，返回 -1，表示没有更多的数据可读（EOF）。

阻塞：如果没有可用的输入，read(byte[] b) 方法将阻塞，直到有数据可读或发生 I/O 错误。

抛出异常：如果发生 I/O 错误，将抛出 IOException。

1.2.3 read(byte[], int, int)

功能：read(byte[] b, int off, int len) 方法用于从输入流中读取一定数量的字节，并将它们存储在字节数组 b 的指定位置。该方法允许读取数据并放入到数组的特定位置，从而提高读取的灵活性和效率。

参数：

  • byte[] b：用于存储从输入流中读取的字节数据的目标字节数组。
  • int off：表示将读取的字节数据存储到数组 b 的起始偏移量（即从 b 的哪个索引位置开始存储读取的数据）。
  • int len：表示要读取的字节数，即尝试从输入流中读取的最大字节数。

返回值：

  • 返回值为一个整数，表示实际读取到的字节数。如果读取到流的末尾，返回 -1。
  • 读取的字节数可能小于请求的字节数 len，这取决于当前输入流中可以提供的字节数。

抛出异常：如果发生 I/O 错误，将抛出 IOException。

1.2.4 write(int)

功能：write(int b) 方法用于将一个字节的数据写入到输出流中。该方法将指定的整数 b 作为字节写入，但实际上只写入该整数的 低 8 位，即 b 的最低一个字节。

参数：int b：要写入的字节数据。

抛出异常：如果发生 I/O 错误，将抛出 IOException。

1.2.5 write(byte[])

功能：write(byte[] b) 方法用于将一个字节数组中的所有字节写入输出流。该方法能够一次性写入多个字节，和单字节写入方法 write(int b) 相比，能够更高效地写入大量数据。

参数：byte[] b：要写入的字节数组。该数组中的所有字节都会被写入到输出流中。

抛出异常：如果发生 I/O 错误，将抛出 IOException。

1.2.6 wirte(byte[], int, int)

功能： wirte(byte[] b, int off, int len) 方法用于将字节数组的一部分写入输出流。这个方法能够更加灵活地处理数据，适合于需要从数组的特定位置开始写入或只想写入部分数据的场景。

参数：

  • byte[] b：要写入的字节数组。

  • int off：指定从数组的哪个位置开始读取数据（偏移量）。

  • int len：要写入的字节数，表示从偏移量 off 开始要写入的字节长度。

抛出异常：如果发生 I/O 错误，将抛出 IOException。

1.2.7 append data

在 Java 中，追加写入（Append Writing）通常是指在文件末尾添加数据，而不是覆盖文件中现有的内容。为了实现文件的追加写入，可以使用 FileOutputStream 类的构造方法中的第二个参数，设置为 true，以便在写入时启用追加模式。

1.2.8 copy data

在 Java 中进行数据复制（copy data）通常涉及将数据从一个流读入（输入流）并写入到另一个流（输出流）。可以使用 FileInputStream 和 FileOutputStream 来进行文件复制操作。这两个类分别用于读取文件和写入文件。为了提高性能，通常使用缓冲流（BufferedInputStream 和 BufferedOutputStream）来减少磁盘 I/O 次数。

1.3 FR/RW 操作

FileReader 类构造函数分析

super() 方法将 FileInputStream 传递给 InputStreamReader，再调用 StreamDecoder 类的 forInputStreamReader() 方法将字节流按照指定的字符编码转换为字符流。

forInputStreamReader 是 StreamDecoder 的一个静态工厂方法，用于创建合适的 StreamDecoder 实例。这个方法需要传入字节输入流、用于同步的锁对象和字符编码，并返回一个可以处理字节到字符转换的 StreamDecoder。（这里先暂时不过多介绍）

super(new FileInputStream(file))：通过调用 super，将 FileInputStream 传递给 FileReader 的父类 InputStreamReader。

所以 FileReader 实际上是通过 InputStreamReader 来实现字节到字符的转换：

  • FileReader 继承了 InputStreamReader，而 InputStreamReader 是字节流到字符流的桥梁。它负责将从 FileInputStream 读取的字节调用 StreamDecoder 类的方法按照指定的字符编码转换为字符。
  • 这意味着，FileReader 只是一个方便的类，用于简化创建 InputStreamReader 和 FileInputStream 的过程，它默认使用系统的字符集来读取文件。

FileWriter 类构造函数分析

super() 方法将 FileOutputStream 传递给 OutputStreamReader，再调用 StreamEncoder 类的 forOutputStreamWriter() 方法将字节流按照指定的字符编码转换为字符流。

forOutputStreamWriter 是 StreamEncoder 的一个静态工厂方法，用于创建合适的 StreamEncoder 实例。这个方法需要传入目标输出流和字符编码，并返回一个可以处理字符到字节转换的 StreamEncoder。（这里先暂时不过多介绍）

FileWriter 通过 super(new FileOutputStream(file)) 调用父类 OutputStreamWriter 的构造函数，而 OutputStreamWriter 是负责将字符流转换为字节流的类。它将字符编码为字节，并将这些字节通过 FileOutputStream 写入文件。

所以 FileWriter 实际上是通过 OutputStreamWriter 来实现字符到字节的转换：

  • FileWriter 继承自 OutputStreamWriter，用于将字符数据写入文件。
  • FileOutputStream 负责处理底层的字节写入操作，因为文件系统只能存储字节数据。
  • FileWriter 通过字符流与字节流的桥梁 OutputStreamWriter 来将字符数据转换为字节，然后使用 FileOutputStream 将字节写入文件。

1.3.1 read(CharBuffer)

功能：read(java.nio.CharBuffer target) 方法用于将字符从文件中读取到 CharBuffer 中。这个方法的工作原理是尽可能多地读取字符，直到缓冲区没有足够的空间为止。读取的字符直接放入 CharBuffer 对象中，缓冲区的当前指针也会随之移动。

参数：CharBuffer target：用于存放从文件中读取字符的 CharBuffer 对象。该缓冲区应处于写入模式。

返回值：返回实际读取的字符数，或者返回 -1， 表示已到达文件结尾。

异常：如果发生 I/O 错误，将抛出 IOException。

1.3.2 read()

功能：read() 方法用于读取单个字符，返回该字符的 int 表示。如果已到达文件结尾，返回 -1。

返回值：读取的字符对应的 int，或 -1 表示文件结尾。

异常：如果发生 I/O 错误，将抛出 IOException。

1.3.3 read(char[])

功能：将字符读入字符数组 cbuf，最多读取 cbuf.length 个字符。

参数：char[] cbuf：目标字符数组，用于存储读取的字符。

返回值：返回实际读取的字符数，或者返回 -1 表示已到达文件结尾。

异常：如果发生 I/O 错误，将抛出 IOException。

1.3.4 read(char[], int, int)

功能：将字符读入到字符数组 cbuf 中，从偏移量 off 开始，最多读取 len 个字符。

参数：

  • char[] cbuf：目标字符数组，用于存储读取的字符。
  • int off：从 cbuf 中开始存储字符的偏移量。
  • int len：要读取的字符最大数量。

返回值：返回实际读取的字符数，或者返回 -1 表示已到达文件结尾。

异常：如果发生 I/O 错误，将抛出 IOException。

1.3.5 write(int)

功能：FileWriter.write(int) 方法用于将单个字符写入到文件中。

参数：int c：要写入的字符。虽然参数类型是 int，但只有该整数的低 16 位（即字符的 Unicode 编码值）会被写入文件。

异常：如果发生 I/O 错误，将抛出 IOException。

1.3.6 write(char[])

功能：FileWriter.write(char[]) 方法用于将字符数组中的字符写入文件。该方法会将整个字符数组的内容一次性写入到文件中。

参数：char[] cbuf：要写入的字符数组。

异常：如果发生 I/O 错误，将抛出 IOException。

1.3.7 write(char[], int, int)

功能：FileWriter.write(char[], int, int) 方法用于将指定字符数组中的一部分字符写入文件。该方法允许选择从字符数组的哪个位置开始写入，以及最多写入多少个字符。

参数：

  • char[] cbuf：要写入的字符数组。
  • int off：字符数组中开始写入的起始位置（偏移量）。
  • int len：要写入的字符个数，从偏移量 off 开始算起。

异常：如果发生 I/O 错误，将抛出 IOException。

1.3.8 write(String)

功能：FileWriter.write(String) 方法用于将指定的字符串写入文件。该方法会将字符串中的所有字符写入到文件中。

参数：String str：要写入文件的字符串。该字符串中的所有字符将被写入到文件中。

异常：如果发生 I/O 错误，将抛出 IOException。

1.3.9 write(String, int, int)

功能：FileWriter.write(String, int, int) 方法用于将指定字符串中的一部分字符写入文件。该方法允许指定从字符串的哪个位置开始写入，以及最多写入多少个字符。

参数：

  • String str：要写入文件的字符串。
  • int off：字符串中开始写入的起始位置（偏移量）。
  • int len：要写入的字符个数，从偏移量 off 开始算起。

异常：如果发生 I/O 错误，将抛出 IOException。
